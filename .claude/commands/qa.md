# /qa — the QA certification gate

This is a deliberate, PI-timed certification gate, **not** a routine pass.

> **PI-INVOKED ONLY.** The PM must never self-trigger this, run it proactively, or nudge toward it each sprint. It fires only when the PI types `/qa`. Certifying a branch "done" is a timing judgment that belongs to the PI; auto-firing would defeat that. (It is a genuine-trigger command like `/release` and `/sprint-close`, with **no** corresponding memory rule — CLAUDE.md §13.9a.)

**Why this exists.** A reviewer tasked with "find problems" always finds problems, so a "clean" can never be *earned* and a "dirty" can never be *trusted* — it is an activity, not a test. This gate converts the fault-hunt into a test with discriminating power, using four mechanisms (pre-registered criteria, materiality grading, per-run calibration controls, convergence) and returns a **three-way verdict** that distinguishes *"target not done"* from *"reviewer not trustworthy."*

## Inputs

- **Target** = `$ARGUMENTS` (`trunk`, `group1`, …). If absent, ask the PI which target.
- **Definition of done** = `docs/qa/<target>.done.md` — the pre-registered, binary, material criteria. **If it does not exist, STOP** and co-write it with the PI first. Pre-registration must precede the review; the reviewer is never allowed to define "done."
- **Seed catalog** = `docs/qa/seed_defects.md` — the known defect classes to plant.

## Protocol

1. **Load & freeze the criteria.** Read `docs/qa/<target>.done.md`, quote it back, and confirm with the PI that nothing has been added or relaxed for this run. The goalposts are fixed before any review begins. **Run the DETERMINISTIC checks the criteria name** — they certify their classes more reliably than an LLM reviewer (the run-#1 / run-#4 lessons), leaving the dispatched reviewers to *judgment* calls:
   - **C11 internal-title consistency** — `debug/qa/check_internal_titles.py` (exact string match → clean PASS/FAIL).
   - **C5 K-label backstop** — `debug/qa/check_k_label.py` (FAILs on any SUSPECT K-tier assertion in the gated **trunk** scope; the **AUDIT** list it prints is advisory non-trunk drift, surfaced for a PI sweep decision, and does **not** fail the gate). This is a *backstop*, not a replacement: the K-prohibition is semantic, so the `claims-reviewer` still enumerates K-sentences (the screen can't anchor a bare "K" without the rule name in-clause).
   - **C13 paper↔test reference integrity** — `debug/qa/check_paper_test_refs.py` (FAILs if any test cited inline in a gated paper is non-existent/renamed/archived; reports matrix-coverage gaps + non-trunk stale refs as advisory). The deterministic guard that the inline test references stay honest against the suite + `docs/claim_test_matrix.md`. `--gate <branch>` per sweep.
   - **C14 paper↔file reference integrity** — `debug/qa/check_file_refs.py` (FAILs if any `geovac/`, `benchmarks/`, or `demo/` — permanent code/artifact — file path cited inline in a gated paper does not resolve; `debug/` refs are advisory, since the transient clean-room dir is pruned by design). Closes the C13 gap (C13 covers `tests/` refs only): the group3 run-4 `geovac/jlo_chi.py` defect was a claim citing a nonexistent code module. `--gate <branch>` per sweep.
   - **C16 retracted-claims / zombie-drift** — `debug/qa/check_retracted_terms.py` (FAILs if a known-retracted claim re-surfaces *without* an inline withdrawal flag; the registry in that file holds every phrase a prior /qa run retired — the withdrawn Pythagorean C₃<1 form, the S⁷ "structural negative" vs its erratum, the Batch-2 false-closure framing, "propinquity"-as-achieved-metric, …). **This is the deterministic answer to the 2026-06-23 lesson:** the recurring zombies were *mechanical* drifts in low-salience regions (status-table cells, docstrings, footnotes, one abstract clause) that judgment-mode LLM review keeps walking past (the withdrawn Pythagorean form survived **three** incomplete sweeps); a grep is exhaustive and zero-variance. **HARD RULE: whenever a /qa run retires/withdraws a claim, ADD its phrase to the `REGISTRY` in `check_retracted_terms.py`** so it can never silently re-surface. `--gate <branch>` per sweep; scans the gated papers AND their contained code/test modules.
   - **C17 headline-number registry** — `debug/qa/check_headline_numbers.py` (FAILs if any DoD §C8 headline-number family appears with a non-canonical value; families carry require-context + exemption windows so different quantities sharing a numeral never false-positive). **The deterministic answer to the 2026-07-02 lesson (7th group4 cert):** the surviving genuine-MATERIAL classes were mechanical — second-locus propagation (Z=1–36 vs 56 ×5 loci) and number-vs-source drift (a "190×" floor vs the cited table's 51×) — which a panel finds one locus at a time at full price and a registry finds exhaustively for ~0. **HARD RULE (mirrors C16): when a run corrects or demotes a headline number, ADD/UPDATE its family in the `REGISTRY`.** Self-test: `tests/test_headline_numbers_check.py`.
   - **C18 project-course duration language** — `debug/qa/check_duration_language.py` (FAILs on historical wall-clock durations attached to the project's course — "N years ago", "units of work", "across N days"; forward effort-estimate vocabulary ("multi-month/-year", "week-scale") is advisory standing debt pending the corpus sweep). **The deterministic answer to the 2026-07-04 lesson:** LLM drafting is unreliable about elapsed project time (the field guide's fabricated "Three years ago the project…"), and this class is pure grep. Sequence language, real dates, version anchors, and unit-free effort vocabulary ("sprint-scale", "deep wall") are fine. Convention: `docs/authoring_conventions.md` rule 12; `--selftest` built in.

   **Run shapes (2026-07-02, PI direction — the cost protocol).** A `/qa` invocation is one of two shapes; choose FIRST, before any dispatch:
   - **FULL run** — everything below, all dimensions, whole-paper enumeration, completeness-critic. Fired at exactly two moments: the **first cert** of a fresh target, and the **final certifying pass**. Only a full run can produce PASS.
   - **DELTA-verification run** — after any FAIL→remediation. Scope = the git diff since the last calibrated run. Deterministic gates still run whole-target (free). LLM review: one reviewer per **affected** dimension only; the changed loci **pasted into the prompt** with context (paste-don't-point); **≥1 seed planted in the diff per dispatched agent** (two if the agent runs below Opus tier); no completeness-critic. Verdict: **CLEAN-DELTA / DEFECTS** — never PASS; **a clean delta is the precondition for firing the full certifying run.** Rationale: text that survived a calibrated panel does not lose verification by sitting unchanged — the deterministic layer (C16+C17 registries) guards the unchanged surface. The group4 arc paid full price (~2.5M tokens) on "this is probably the one" four times; a delta cycle costs ~0.5M.

2. **Build an isolated seeded copy.** `git worktree add ../geovac-qa-seed-<target> -b qa-seed-<target>` (or reuse a temp branch). **Seeds NEVER touch the real corpus** — all planting happens in the worktree, which is deleted at the end.

3. **Plant calibration controls (blind).** From `docs/qa/seed_defects.md`, plant K (≈4–6) *realistic* defects from our actual §3 error history (fabricated arXiv ID, circular/tautological test, κ-style overclaim, stale-title drift, hard-prohibition touch, …) into the worktree copy of the target. **Cover every gating dimension:** plant at least one seed catchable by *each* dimension's reviewer type (a code-defect for `code-reviewer`, a prose-defect for `claims-reviewer`, a citation-defect for `citation-reviewer`, a synthesis-defect for the synthesis `claims-reviewer`), so no dimension that gates the verdict goes uncalibrated this run. Also designate M (≈4–6) **known-good** claims that are verified correct and must NOT be flagged material. Record the answer key — {file, location, defect-class, expected-catcher} — in `debug/qa/<target>_seed_key.json`. The reviewers must be blind to which items are seeded. **Model-tiered agents get TWO seeds** (see the tiering rule in step 4): calibration resolution is the safety net that makes the cheaper tier trustworthy.

   **Per-chunk seeding when a dimension is consolidated (the step-4 granularity rule).** Calibration is per-AGENT, not per-dimension: a chunk-agent that received no seed is *uncalibrated*, so its "clean" is untrustworthy ⇒ INCONCLUSIVE for the papers in that chunk. So when a dimension is chunked (citation/claims covering several papers per agent), seed **≥1 catchable defect per (dimension × chunk)**, and plant it in a **non-first paper of the chunk** — a seed in the chunk's first paper lets a consolidated agent pass calibration by reading only paper 1 and then sampling the rest (defeating the point of consolidation). For a whole-group run this means more seeds (≈ one per chunk-agent), spread so every dispatched agent's discrimination is actually exercised.

4. **Dispatch the reviewers across EVERY dimension — fresh, blind, pointed at the worktree.** The target's criteria partition into independent **review dimensions**, and a single `/qa` run MUST exercise *all* of them in one invocation. **Passing one dimension never excuses skipping another, and the run does not terminate early when a dimension comes back clean** — the dimensions are independent axes, not sequential gates, so you want the full defect picture from all of them every run. For the trunk the dimensions, their reviewer types, and the criteria they gate are:
   - **Code / test-backing** (C1–C2) — **`code-reviewer`**, one per paper that has backing tests: claim ↔ test ↔ code; *RUN the tests*; does each genuinely *prove* its claim (not tautological / false-positive / weaker than prose)?
   - **Paper claims / prose** (C3, C5, C6, C8) — **`claims-reviewer`** (granularity per the scaling rule below): does any claim assert more than its backing? is the tier inline? are the hard prohibitions intact?
   - **External citations** (C4) — **`citation-reviewer`** (granularity per the scaling rule below): do external cites resolve and say what we attribute?
   - **Synthesis faithfulness** (C9) — **`claims-reviewer`**, one on the branch synthesis: zombie/descoped claims? status overstatement? faithful to the papers it summarizes? (This is a *separate* dispatch from the per-paper claims review — the synthesis is its own document.)
   - **Deterministic** (C10 compiles, C11 internal titles) — the scripts in step 1, not an LLM reviewer.
   Each LLM reviewer reviews **against the pre-registered criteria**, classifies findings MATERIAL vs NIT, and attaches a counterfactual to each MATERIAL finding ("if fixed, would a *target result* change?"). They are fresh agents that never saw each other or the discovery-mode rationale (§9 QA principle 2). **If a gating dimension cannot be exercised this run, the target verdict is INCONCLUSIVE — never PASS (step 7).**

   **Reviewer granularity & scaling to a whole group (the 2026-06-23 right-sizing).** The dimensions are NOT symmetric in how much per-paper depth they need, so set agents-per-dimension by depth-need, not uniformly (the old "one reviewer per paper per dimension" over-spawns — a 5-paper batch was ~15+ agents). Two axes: *which* dimension, and *how many papers per agent (chunk size)*. This is exactly what lets ONE `/qa` invocation cover an entire group, not just a 4–5-paper batch:
   - **Deterministic** → **whole group, always, FIRST.** The step-1 scripts run `--gate <group>` over every paper in seconds; this is the cheapest, most exhaustive whole-group pass we have, and most cross-paper drift (zombie phrases, title/ref/citation-format consistency) dies here for ~0 tokens. Foreground it before any LLM dispatch.
   - **Code / test-backing** → **1 paper per agent** (depth + must RUN the suite + holds its test output; consolidating both blows context AND dilutes the tolerance-scrutiny that finds the defects). One per paper-that-has-tests — this dimension does **not** consolidate, and it is the one that bounds the fan-out size.
   - **External citations** → **chunk ~5–6 papers per agent** (mechanical metadata verification; helped by seeing shared apparatus once; double-backstopped by C15 + C16).
   - **Paper claims / prose** → **chunk ~3–4 papers per agent** (depth-bounded; the cross-paper tier/zombie drift it used to carry is now absorbed by C16, so per-paper is no longer required) + the enumeration mandate below.
   - **Synthesis** → 1; **completeness-critic** → 1.
   So a ~14-paper group fans out to roughly *(papers-with-tests)* code + ~4 claims + ~3 citation + 1 synthesis + 1 critic in ONE invocation — bounded by the per-paper code count, **not** by papers×dimensions. Dispatch a large fan-out in **waves** (the concurrency cap serializes anyway, and big single fan-outs collide with rate limits). **Tractability rule:** if a single chunk-agent's papers won't fit at depth (too many bibitems / too much prose), shrink the chunk — never let a consolidated agent *sample* a paper it was supposed to cover (sampling is the exact coverage-variance failure this whole doctrine fights). **First-cert vs later cycles (superseded 2026-07-02 by the run-shapes rule in step 1):** **batch the FIRST cert of a fresh group** (smaller remediation cycles — you do not want 12 defects across 14 papers surfacing in one giant FAIL); after any FAIL→remediation, run a **DELTA-verification pass** (diff-scoped, per step 1), and fire the whole-group **FULL certifying run only once a delta comes back clean**.

   **Model tiering (2026-07-02, PI direction).** Citation and code dimensions default to **Sonnet-class** reviewers (`model: "sonnet"` on dispatch); claims and synthesis stay **Opus-class**. Calibration converts model adequacy into a per-run measurement: a tiered agent that misses its seed is de-calibrated → re-dispatch that dimension on the Opus tier (the run-6 P16 recovery pattern). Tiered agents carry two seeds (step 3).

   **Terser reporting contract (2026-07-02).** The enumeration mandate governs the READING, not the report. Reviewer output = defects + two-way upgrades + a compact coverage checklist (which regions were enumerated). Do NOT reproduce per-item verdict tables for sound items — soundness is evidenced by the checklist, the seeds, and PM spot-checks.

   **Enumeration-forced reading (the 2026-06-23 lesson).** The recurring misses across the Batch-2/Batch-3 cert arcs were *not* subtle judgment calls — they were mechanical drifts in **low-salience structured regions** (status/catalogue-table rows, docstrings, footnotes, one abstract clause) that a top-to-bottom judgment-mode read systematically under-weights (a synthesis reviewer literally read past a planted abstract seed while finding a harder zombie). So every reviewer prompt MUST demand **exhaustive enumeration**, not sampling, of those regions: *"enumerate EVERY row of every status/catalogue table, EVERY bibitem, EVERY occurrence of the branch's tier-terms (state-space GH vs propinquity, derived vs match, withdrawn-vs-live), and EVERY docstring of the backing module — one verdict per item; do not sample."* Prefer **one enumeration-forced reviewer per dimension over two vaguely-prompted ones** — duplicate agents with the same prompt share blind spots, so diversity/precision of instruction beats redundancy (and costs less). **Internal-consistency forcing (the delta-1 lesson, 2026-07-02, same family as the run-4 K-hedge lesson):** for every sentence containing a quantitative ordering / dominance / superlative claim, the reviewer must check each clause against every other clause of the same sentence and the numbers alongside it, and must NOT construct charitable readings — a self-contradicting sentence is a defect; quote the contradicting clauses side by side. (A delta claims-reviewer rationalized a planted "126 is the dominant gap" past the same sentence's list of four larger gaps; the sharpened re-dispatch caught it immediately.)

   **Completeness-critic pass (cheap coverage closure — FULL runs only; skip on delta runs).** After the panel returns, dispatch ONE final agent whose only job is to name un-verified regions: *"which tables / footnotes / docstrings / status-strings / claims did NO reviewer quote? list the coverage gaps."* This is one agent's cost aimed at the exact failure mode (coverage variance), instead of doubling the panel; re-dispatch a focused reviewer for any gap it surfaces. **Reserve a full find→fix→re-scan loop-until-dry for the *final* pre-certification convergence run only** — it is the most thorough but most expensive shape (each cycle ≈ a full panel); the deterministic gates (incl. C16) + the completeness-critic capture most of its benefit every run for far less.

   **Path-pin (mandatory — the group3 bite-2 S8 miss).** Every reviewer prompt MUST give the ABSOLUTE **worktree** path (`../geovac-qa-seed-<target>/…`) for the files it reads, and MUST explicitly forbid reading the real corpus (`papers/…` under the project root). A reviewer that silently re-reads the real (unseeded) corpus misses every planted defect in its dimension and **de-calibrates the run with no error at all** — a synthesis `claims-reviewer` did exactly this:\ it first caught the planted synthesis seed, then *reversed* the catch after re-reading the live (unseeded) file. Before trusting a reviewer's calibration score, spot-check that its quoted line numbers/text came from the worktree, not the real corpus.

5. **Score calibration — certify the reviewers (against the answer key).**
   - **Sensitivity:** fraction of planted defects the panel collectively caught. A planted class no reviewer caught ⇒ the panel is *blind to that class*.
   - **Specificity:** any known-good control flagged MATERIAL is a *false positive*.
   - A panel that misses planted defects or manufactures faults on the controls is **not calibrated for this run**.

6. **Materiality + convergence on the genuine findings.** Set aside findings at seed/control locations. For the rest, the PM **verifies every MATERIAL finding against primary text/code before accepting it** (§9 reconcile rule). Converge: take the union of *verified* material findings across the independent reviewers.

7. **Emit the verdict — per dimension, then rolled up (AND across dimensions).** Score *each* dimension separately (was it exercised? calibrated? clean?), then combine. The target's verdict is the AND over all gating dimensions:
   - **PASS** — *every* gating dimension was **exercised**, **calibrated** (caught its plants, zero false positives on its controls), and **clean** (zero verified material defects), and independent reviewers converged. *Trustworthy clean across all dimensions.*
   - **FAIL** — every gating dimension calibrated **and** ≥1 verified material defect in *any* dimension. *Target not done* — list each defect with its **dimension** and failing criterion.
   - **INCONCLUSIVE** — any gating dimension was **not exercised**, *or* was exercised but **not calibrated** (missed its plants, or flagged a known-good control). *The verdict is untrustworthy this run.* Name the dimension and what failed. **An unexercised gating dimension is INCONCLUSIVE — never silently downgraded to a "PASS on the exercised dimensions" with a ceiling note.** Fix and re-run.

   Report the verdict as a **per-dimension scorecard** (dimension → exercised / calibrated / clean) plus the roll-up, so the PI sees which axes actually carried the verdict and which (if any) forced INCONCLUSIVE.

8. **Clean up.** `git worktree remove` the seeded copy. Confirm no seed reached the real corpus.

## Output to the PI

- **Verdict:** PASS / FAIL / INCONCLUSIVE.
- **Calibration scorecard:** sensitivity (plants caught / planted, per defect-class) and specificity (false positives / controls) — so the PI can see the panel's *demonstrated* discrimination this run.
- **Material findings** (if FAIL): each verified against primary text, with its failing criterion and counterfactual.
- **Nit log:** non-material findings (logged, do not block).
- **Honest ceiling:** "PASS" means *survived the calibrated detectors for the defect classes in the seed catalog and the pre-registered criteria* — not provably perfect. Name any criterion or defect-class that could not be exercised this run.

## Hard rules
- **PI-invoked only.** PM never self-triggers, runs proactively, or suggests it each sprint.
- Pre-registered criteria are **frozen before** the review — no goalpost-moving in either direction.
- Seeds live only in the worktree; never commit/leak them; always remove the worktree.
- A reviewer's verdict is trusted only **after** it passes calibration (caught the plants, passed the controls). An uncalibrated panel ⇒ INCONCLUSIVE, never PASS.
- **All dimensions every run.** One `/qa` run exercises *every* review dimension the criteria reference — code/test-backing, paper-claims, citations, **and** synthesis — in a single invocation. Dimensions are independent: a clean one never short-circuits another, the run never stops early on a pass, and an **unexercised gating dimension forces INCONCLUSIVE, not PASS** (the run-#2 lesson: a "PASS on the exercised dimensions" hid real defects in the two dimensions that were skipped). The verdict is the AND across all dimensions.
